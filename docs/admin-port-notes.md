## 移植時に修正が必要（モックはプレースホルダ/バグ）
- [x] 整備アラートの「整備登録」ボタン：**整備専用モーダル**を実装（対象車両／整備区分〈定期点検・故障修理・タイヤ・オイル・バッテリー・自賠責更新・車検・その他〉／内容メモ／入庫・出庫予定／費用／担当）。※入庫期間はガントの「整備」帯に反映する旨を注記（連携は実データ移植時）
- [ ] メール配信「＋新規配信を作成」モーダル：内容はOKだが**表示範囲が崩れている**（チェックボックスとラベルが縦潰れ）。レイアウト修正
- [ ] 複数階層メニューは「タイトル右にサブタブ」を原則（商品マスタの子〈クラス/車種/車両〉等も要確認）
- [ ] **商品マスタ**：画面側の「クラス/車種/車両」切替が分かりづらい。エンティティ切替（クラス/車種/車両）をタイトル右の主タブに、表示モード（一覧/ボード/ガント）は副次に。実ページ移植時に明快なタブ構成へ再設計
- [ ] **料金設計**：「海外向け設定」を独立ページに分離 → タブを **クラス料金 / 車種料金 / 海外向け** の3構成に（現状は下層に混在）
- [x] サイドバー「空き通知」→「**空き通知リクエスト**」（管理者に通知が届いた状態と誤認するため）※モック反映済

## 実React実装の進捗（画面単位）
- [x] **ダッシュボード**（`/admin`）
- [x] **予約管理**（`/admin/reservations`）— 一覧／ボード／ガントの3ビュー、契約書印刷タスク（未印刷カウント連動）、検索・店舗・状態フィルタ、CSV/契約書モーダル。詳細は `/admin/reservations/[id]`（KEYBOX発行・返却承認・契約書・延長・キャンセルのモーダル）。
  - 共有部品を追加：`components/admin/Gantt.tsx`（今日基準・範囲切替・横スクロールレバー。日付はmount後にクライアント計算＝SSR不一致回避）、`components/admin/AdminModal.tsx`、`lib/adminGanttData.ts`（75台を決定的生成・店舗別→cc昇順）。バイクスケジュール/商品マスタのガントで再利用予定。
  - ガント範囲は据え置き（既定 30日前〜90日後）。バイクスケジュールは `compact` で1画面に収める予定。
  - 免許未確認の「確認」は当面 予約詳細へ誘導。会員情報リンクも会員管理実装まで無効。
- [x] **承認待ち**（`/admin/approvals`）— 種別チップ絞り込み（すべて/免許証/返却完了/事故報告/写真）、行から各詳細へ。
- [x] **免許確認**（`/admin/license`）— 日本/海外チップ、目視確認モーダル（日本＝番号/種別/確認結果、海外＝発行国/書類アップロード状況/確認結果、「その他」で理由入力を表示）。
- [x] **返却確認**（`/admin/approvals/return/[id]`）・**事故報告**（`/admin/approvals/accident/[id]`）— 承認待ちから遷移する詳細。承認/差し戻し/NOC・受理/通知/保険連絡先/却下のモーダル。
- [x] ダッシュボード「今日のタスク」CTA を各実ページへリンク（印刷/受取→予約管理、返却/事故→承認待ち、免許→免許確認）。
