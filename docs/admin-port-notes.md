# 管理画面 実ページ移植メモ（admin-port）

参照モック: `public/mock/admin-proto-v2.html`（narikiyo087/YASUKARI の admin プロトv2を取り込み・調整済み）
方針: 既存 `/admin` を壊さず `/admin/v2` に段階移植 → 完成後に差し替え。実データは一旦ダミー。

## 反映済み（モック側で調整済み・移植時にそのまま反映）
- [ ] ページヘッダー：タブ名を繰り返すだけの副題を削除（KEYBOX / バイクスケジュール / 分析）
- [ ] KEYBOX：手動「発行」タブ・発行フォーム・サイドバー子項目を廃止（予約時に自動発行）→ **実行ログ / 再発行** のみ
- [ ] 「スケジュール」→ **「バイクスケジュール」**（ナビ＋タイトル）
- [ ] バイクスケジュールのガント：**1画面に収める＋縦スクロール**（予約管理のガントは据え置き）
- [ ] ガント車両行：**店舗別 → 排気量(cc) 昇順**でソート

## 移植時に修正が必要（モックはプレースホルダ/バグ）
- [ ] 整備アラートの「整備登録」ボタン：今は汎用モーダル(`M('edit')`)。**整備専用モーダル**へ（対象車両／整備区分〈定期・故障・タイヤ・オイル等〉／内容メモ／入庫・出庫予定／費用／担当）
- [ ] メール配信「＋新規配信を作成」モーダル：内容はOKだが**表示範囲が崩れている**（チェックボックスとラベルが縦潰れ）。レイアウト修正
- [ ] 複数階層メニューは「タイトル右にサブタブ」を原則（商品マスタの子〈クラス/車種/車両〉等も要確認）
- [ ] **商品マスタ**：画面側の「クラス/車種/車両」切替が分かりづらい。エンティティ切替（クラス/車種/車両）をタイトル右の主タブに、表示モード（一覧/ボード/ガント）は副次に。実ページ移植時に明快なタブ構成へ再設計
- [ ] **料金設計**：「海外向け設定」を独立ページに分離 → タブを **クラス料金 / 車種料金 / 海外向け** の3構成に（現状は下層に混在）
- [x] サイドバー「空き通知」→「**空き通知リクエスト**」（管理者に通知が届いた状態と誤認するため）※モック反映済
