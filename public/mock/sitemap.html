ヤスカリ サイトマップ全体図

顧客サイト（日本語／英語）と管理画面の「あるべき構成（To-Be）」。ここで全体を固めてから、1ページずつ実装を調整します。

主要CTA（予約導線） 認証が必要 統合・新設 管理画面 通常ページ

顧客サイト（日本語 / 英語は同構造）

設計原則：主導線（車種→予約）を最短に／情報は5グループに集約／開発用ページは公開ナビから隔離

トップ→ 車種一覧→ 車種詳細→ 空き確認→ 予約 step1–3 → 完了

1バイクを探す

  • トップ /
  • 車種一覧 /products
  • ジャンル別フィルタ/t/genre を統合
  • 車種詳細 /products/[code]

2予約する

  • 空き確認 /reserve/…/availability
  • 予約フロー step1 → 2 → 3CTA
  • 予約完了 /reserve/flow/complete
  • 貸渡約款 /…/rental-bike-terms
  • 決済完了 /payment-info/complete

3はじめての方

  • はじめてガイド /beginner改善版済
  • 料金・補償 /pricing・/insurance
  • 店舗案内 /stores（足立/三ノ輪）
  • /guide は統合・廃止統合

4会員（要ログイン）

  • ログイン /login1本化
  • 会員登録 /signup → /register
  • マイページ /mypage
  • レンタル詳細・延長 /mypage/rentals/[id]
  • 過去の予約・会員情報・お知らせ

5サポート

  • ヘルプ/FAQ /help
  • 問い合わせ /contact＋チャット
  • 会社情報 /company

6お知らせ・ブログ

  • お知らせ /news
  • ブログ /blog3系統を統合
  • blog_for_custmor/customer_blog/rental_bike を集約

法法務（フッターのみ）

  • 利用規約 /terms
  • プライバシー /privacy
  • 特商法 /tokusyouhou
  • 貸渡約款 /rental-bike-terms

隔公開ナビから隔離

  • /register/test（開発用着地・要撤去）
  • /test・/chat-design・/source
  • /monitor・/wait・/maintenance
英語版 /en：現状は日本語版とほぼ全ページ二重化（HeaderEn/FooterEn 等も別実装）。To-Beは日英で構造を完全一致させ、差分はテキストと価格ロジックだけに閉じ込める。新デザインは日本語版を正として作り、英語版に翻訳適用。

管理画面 /admin/dashboard

設計原則：「今日やること」を最上位に／マスタ設定と日次運用を分離／承認待ちを1箇所に集約（現状は平坦30項目超）

①今日の運用（Daily Ops）

  • ダッシュボード（本日の予約/返却/承認待ちサマリ）運用サマリ化
  • 予約管理 reservations
  • 承認待ち（統合）4種を集約
  • 　└ 免許証 / 事故報告 / 返却完了 / 写真
  • KEYBOX（発行/再発行/ログ）
  • バイクスケジュール

②会員・連絡

  • 会員管理 members
  • メール送信履歴 / テスト送信
  • メルマガ設定
  • チャットボット（QA / 問い合わせ）

③商品マスタ（Catalog）

  • クラス bike-classes（＋月額料金）
  • 車種 bike-models（＋レンタル料金）
  • 車両 vehicles
  • 用品・オプション accessories

④販促・カレンダー

  • 新着情報/告知バー announcements
  • ブログ blog
  • 休日管理 / ハイシーズン
  • クーポン coupon-rules

⑤分析（Analytics）

  • 予約 / 会員 / 車両 / 車種

⑥サイト設定

  • サイト公開状態 site-visibility
管理画面の要点：① 承認待ち（免許/事故/返却/写真）を1グループに集約して見落とし防止／② サイドバーを6グループの折りたたみに／③ ダッシュボード初期表示を「本日の予約・返却・承認待ち件数」の運用サマリに。

フランチャイズ前提（店舗が増えたとき）

本部（ケイジェット）＝仕入れ・整備・マスタを一括／現地オーナー＝自店の運営。ロール＋店舗スコープが土台。

トップ（全店を広く）→ 車種詳細（取扱店舗・空き）→ 仮登録（メアド+PW+国）→ 受取店舗を確定→ 予約 step1–3

本部ケイジェット（全店）

  • 車種・クラスのマスタ
  • 料金ベース（基本は全店統一）
  • 車両の仕入れ登録・整備手配
  • 全店の予約・会員・分析
  • 店舗マスタ / スタッフ・ロール管理HQのみ新設
  • サイト公開設定

現地店舗オーナー（自店のみ）

  • 自店の予約管理
  • 自店の承認（免許・事故・返却）
  • 自店の KEYBOX
  • 自店の休日・営業設定
  • 店舗限定クーポン / 用品の表示ON/OFF
  • ※ マスタ・料金ベース・全社分析は不可閲覧のみ

基盤土台（本丸・要追加見積もり）

  • ロール＋店舗スコープ（自動フィルタ）
  • 店舗（store）を第一級データに
  • 各一覧に店舗フィルタ（予約/承認/車両/KEYBOX/クーポン）
  • 会員は全社共通を推奨（履歴は店舗単位で可視）
2つの土台：(A) データの店舗化（車両/予約/用品/休日/クーポンに店舗を紐付け）と (B) 権限の店舗化（現地オーナーが自店だけ見える仕組み）。見積もりに載っているのは主に(A)。(B)＝権限基盤は未計上＝Phase 1で追加見積もりが必要。
